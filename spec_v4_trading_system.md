# 量化策略分析平台 v4.3 — 交互式网页规格文件

## 目标

构建一个纯前端单页交互式量化策略分析网页，用户上传复权后 Excel 数据、输入投资资金，选择策略模型并配置买入/卖出/仓位规则后，系统自动执行回测模拟，输出策略图表、绩效指标、数据诊断、持仓盈亏等完整结果。

---

## 技术栈

| 组件 | 方案 |
|------|------|
| 文件解析 | SheetJS (xlsx) CDN |
| 图表渲染 | ECharts 5.x CDN |
| 架构 | 纯前端单 HTML 文件，无后端依赖 |
| 页面结构 | 卡片式布局，响应式设计 |

## 页面结构

页面分为**配置区**（上方）和**输出区**（下方），点击"开始分析"后输出区填充结果。

### 配置区

#### ① 数据导入区
- 拖拽/点击上传 Excel (.xlsx) / CSV 文件
- ⚠️ 醒目提示：**请上传复权后的数据**（前复权/后复权均可），包含日期、开盘价、收盘价、最高价、最低价、成交量列
- 列名自动匹配（支持中英文：日期/trade_date、开盘价/open、收盘价/close、最高价/high、最低价/low、成交量/volume）
- 上传后展示"数据预览（前5行）"
- 自动解析日期并排序
- 日期范围选择器（1-5年可选，默认5年）

#### ② 数据诊断（紧跟数据导入）
上传数据后自动计算并展示，4个分类 × 4项指标 = 16项：

| 分类 | 指标 |
|------|------|
| 📋 数据概览 | 总交易日数、日期范围、价格区间、区间涨跌幅 |
| ✅ 数据质量 | 缺失值、OHLC逻辑错误、异常波动日(\|涨跌幅\|>5%)、预期休市缺口 |
| 📈 收益率特征 | 日均涨跌幅、日波动率、年化波动率、最大单日涨/跌幅 |
| 📊 量价关系 | 日均成交量、最大成交量日、涨跌比、连涨/连跌最大天数 |

#### ③ 资金输入区
- 输入框：投资总资金金额（单位：元，默认 100,000）
- 回测后更新显示：可用资金、已用资金、总资产净值

#### ④ 策略模型选择
- 下拉选择框列出所有注册模型
- 参数面板根据模型动态渲染，**用户自行输入参数值**
- 已实现：双均线交叉 (MA Crossover) — 用户输入短期均线周期、长期均线周期
- 预留占位：MACD、布林带、RSI（显示"待实现"）

#### ⑤ 买入管理
分为买入信号和趋势过滤两个子模块，**每个条件独立可勾选开启/关闭**：

**买入信号**（多选，OR 关系 —— 任一选中条件触发即视为买入信号）

| 勾选 | 条件名称 | 当前状态 | 说明 |
|------|---------|---------|------|
| ☑ | 金叉 | **已实现** | 短期均线上穿长期均线 |
| ☐ | MACD金叉 | 待实现 | — |
| ☐ | RSI超卖反弹 | 待实现 | — |
| ☐ | 成交量突破 | 待实现 | — |

**趋势过滤**（多选，AND 关系 —— 所有选中条件同时满足才放行）

| 勾选 | 条件名称 | 当前状态 | 说明 |
|------|---------|---------|------|
| ☑ | 价格在长期均线之上 | **已实现** | close > MA_long |
| ☑ | 短期均线朝上 | **已实现** | MA_short[t] > MA_short[t-1] |
| ☐ | 放量过滤 | 待实现 | — |
| ☐ | 波动率过滤 | 待实现 | — |

> **买入触发逻辑**：
> 1. 买入信号区：遍历所有勾选条件，**任一**满足 → 买入信号 = true
> 2. 趋势过滤区：遍历所有勾选条件，**全部**满足 → 趋势过滤 = true  
> 3. 最终：买入信号 AND 趋势过滤 → 触发买入
>
> 用户可组合：例如只勾"金叉"不勾任何过滤 → 裸金叉策略；勾"金叉"+"价格在长期均线之上"+"短期均线朝上" → 过滤策略

#### ⑥ 卖出管理
分为止损和止盈两个子模块，**每个条件独立可勾选开启/关闭**：

**止损**（多选，OR 关系 —— 任一选中条件触发即执行卖出）

| 勾选 | 条件名称 | 当前状态 | 说明 | 用户可调参数 |
|------|---------|---------|------|-------------|
| ☑ | ATR止损 | **已实现** | 止损价 = 入场价 − ATR倍数 × ATR(周期)；每个子仓位独立止损价（入场时锁定）；当日收盘价 ≤ 止损价 → 触发 | ATR周期(默认14), ATR倍数(默认2) |
| ☐ | 追踪止损 | 待实现 | — | — |
| ☐ | 固定百分比止损 | 待实现 | — | — |

> 当用户勾选 ATR止损时，ATR 周期和 ATR 倍数的输入框才显示

**止盈**（多选，OR 关系 —— 任一选中条件触发即执行卖出）

| 勾选 | 条件名称 | 当前状态 | 说明 | 用户可调参数 |
|------|---------|---------|------|-------------|
| ☑ | 死叉止盈 | **已实现** | 短期均线下穿长期均线 | 止盈方式：全部平仓 / 减仓FIFO(平最早那笔) |
| ☐ | 固定目标收益率 | 待实现 | — | — |
| ☐ | ATR倍数止盈 | 待实现 | — | — |
| ☐ | 时间止损 | 待实现 | — | — |

> 当用户勾选死叉止盈时，止盈方式的下拉选择框才显示

**卖出触发逻辑**：
1. 止损区：遍历所有勾选条件，**任一**满足 → 触发止损卖出
2. 止盈区：遍历所有勾选条件，**任一**满足 → 触发止盈卖出
3. 同一日先检查止损、再检查止盈
4. 用户可组合：例如只勾 ATR止损 不勾任何止盈 → 纯止损策略；只勾死叉止盈 → 纯信号止盈策略

#### ⑦ 持仓及仓位管理

**仓位规则**
- [✓] 1%风险规则：加仓头寸规模 = 账户总资金 × 风险比例 ÷ (入场价 − 止损价)
  - 风险比例：用户输入，默认 1%
  - 总资金以用户输入的初始资金为基准（不复利）
  - 头寸按手取整（A股 100 股/手）
- [ ] 待补充：固定仓位、凯利公式、金字塔加仓…

#### ⑧ 回测设置
- **成交价选项**：用户选择买卖成交价
  - **[✓] 当日收盘价**（默认）：信号触发当日以收盘价成交
  - **[✓] 次日开盘价**：信号触发当日检测，次日以开盘价成交
    - 次日开盘价成交时，止损价仍以信号触发日的 ATR 计算
    - 隔夜缺口止损：次日开盘若已跌破止损价，以开盘价卖出

**[开始分析] 按钮** — 点击后执行回测，输出区填充结果

### 输出区

#### ⑨ 绩效指标面板（图表上方）
- 4个核心指标卡片：
  - **累计回报**：回测期间总收益率 = (期末总资产 − 初始资金) / 初始资金 × 100%
  - **最大回撤 (MDD)**：总资产净值曲线峰值到谷底最大跌幅
  - **夏普比率**：年化风险调整收益（无风险利率 3%）
  - **总交易次数**：完整买入-卖出循环次数
- 涨红跌绿（A股惯例）

#### ⑩ ECharts 策略图表
- 主图：收盘价走势线 + MA短期线 + MA长期线
- 标注层：
  - ▲ 红色三角 = 金叉（买入信号）
  - ▼ 绿色三角 = 死叉（卖出信号）
  - ● 蓝色圆点 = 实际买入成交点
  - ● 紫色圆点 = 实际卖出成交点（止损/止盈）
  - 虚线 = 持仓期间止损价跟踪线
- 支持缩放（dataZoom）、tooltip 悬浮查看 OHLC + MA + ATR

#### ⑪ 持仓表 + 交易记录

**当前持仓表**（回测最后一天的状态）

| 列 | 说明 |
|----|------|
| 建仓日期 | 买入信号触发日 |

| 持仓盈亏 | (当前价 − 入场价) × 数量 |

- 汇总行：总持仓数、总市值、总盈亏、总盈亏%
- 账户概览：初始资金、可用资金、持仓市值、总资产净值、总收益率

## 回测引擎 (Backtest Engine)

回测引擎在模型 `analyze()` 完成后运行，逐日遍历数据模拟交易：

```
初始化: cash = 用户输入资金, positions = [], trades = [], pendingOrders = []
读取用户勾选的条件配置:
  enabledBuySignals   = { golden_cross: true/false, ... }
  enabledFilters      = { above_long_ma: true/false, short_ma_rising: true/false, ... }
  enabledStopLoss     = { atr_stop: true/false, ... }
  enabledTakeProfit   = { death_cross: true/false, ... }

for each day t in data:
  0. 处理隔日委托（仅"次日开盘价"模式）:
     a. 执行上日 pendingOrders 中的买入/卖出委托（以当日开盘价成交）
     b. 检查隔夜缺口: if open[t] ≤ any position.stopLoss → 以开盘价卖出

  1. 计算当日指标 (MA_short, MA_long, ATR)

  2. 检查现有持仓的退出条件（先卖后买）:
     === 止损（遍历已勾选的止损条件，任一触发即执行）===
     if enabledStopLoss.atr_stop:
       for each position:
         if close[t] ≤ position.stopLoss → 触发 ATR止损
           - 当日收盘价: 以 close[t] 卖出
           - 次日开盘价: 加入 pendingOrders

     === 止盈（遍历已勾选的止盈条件，任一触发即执行）===
     if enabledTakeProfit.death_cross AND 死叉信号 at day t:
       按用户选择的止盈方式执行:
         - 全部平仓: 卖出所有持仓
         - 减仓FIFO: 只卖出最早建仓的那笔

  3. 检查买入条件:
     === 买入信号（遍历已勾选的信号条件，OR 关系）===
     buySignal = false
     if enabledBuySignals.golden_cross AND 金叉 at day t:
       buySignal = true

     === 趋势过滤（遍历已勾选的过滤条件，AND 关系）===
     trendOk = true  // 若没有勾选任何过滤条件，默认通过
     if enabledFilters.above_long_ma AND NOT (close[t] > MA_long[t]):
       trendOk = false
     if enabledFilters.short_ma_rising AND NOT (MA_short[t] > MA_short[t-1]):
       trendOk = false

     === 若 buySignal AND trendOk → 叠加建仓 ===
       - 成交价: 当日收盘价 → entry = close[t]
                 次日开盘价 → 加入 pendingOrders, entry = open[t+1]
       - 止损价 = entry − ATR倍数 × ATR[t]（以信号日 ATR 计算）
       - 头寸规模 = floor(总资金 × 风险比例 / (entry − 止损价) / 100) × 100
       - 若可用资金充足 → 买入/加入委托

  4. 更新持仓盈亏 (mark-to-market at close[t])

计算绩效:
  - 累计回报 = (期末总资产 - 初始资金) / 初始资金 × 100%
  - 最大回撤 = 总资产净值曲线的 max(峰值到谷底跌幅)
  - 夏普比率 = (日均收益 - rf/243) / 日收益标准差 × √243
  - 总交易次数 = 完整买入-卖出循环数
```

**注意事项：**
- 买卖均以**用户选择的成交价**成交：
  - 当日收盘价：信号触发当日收盘价执行
  - 次日开盘价：信号触发当日检测，次日开盘价执行，附带隔夜缺口止损检查
- 先检查卖出、再检查买入（同一日不会既卖又买同一标的）
- 头寸按手取整（A股 100 股/手）
- 叠加建仓：已有持仓时金叉仍可加仓，每个子仓位独立维护止损价
- 死叉止盈方式由用户选择：全部平仓 或 减仓FIFO
- 总资金以初始输入为基准，不复利

## 数据流

```
上传Excel → SheetJS解析 → 列名匹配 → 日期排序 → 日期范围裁剪
  → 数据诊断(②)
  → 用户配置:
      资金(③) + 模型参数(④)
    + 买入信号勾选(⑤☑金叉/☐MACD金叉/...) + 趋势过滤勾选(⑤☑价格>MA_long/☑MA_short朝上/...)
    + 止损勾选(⑥☑ATR止损 + 参数) + 止盈勾选(⑥☑死叉止盈 + 止盈方式)
    + 仓位规则(⑦) + 成交价(⑧)
  → [开始分析]
  → 模型 analyze() 计算指标+信号+ATR
  → 回测引擎 (按用户勾选的条件执行)
  → 输出: 绩效指标(⑨) + ECharts图表(⑩) + 持仓表+交易记录(⑪)
```

## v4.3 核心变更 vs v4.2

1. **买入管理**：从静态"已启用"标签 → 改为**真实复选框**，每个条件独立勾选
   - 买入信号 OR 关系，趋势过滤 AND 关系
   - 未实现的条件灰色显示，勾选框禁用
2. **卖出管理**：同理改为复选框
   - 勾选 ATR止损 → 显示 ATR 周期/倍数输入框
   - 勾选死叉止盈 → 显示止盈方式下拉框
   - 未勾选的条件在回测中跳过
3. **回测引擎**：运行时读取用户勾选的配置，动态决定检查哪些条件
4. **UI 交互**：条件参数按勾选状态动态显隐，减少界面视觉噪音
